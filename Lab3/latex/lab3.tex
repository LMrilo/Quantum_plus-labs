\begin{warningbox}
\textbf{What is an Oracle?}\\
In computer science, an Oracle is a "Black Box". We can give it an input $x$ and it returns an output $f(x)$, but we \textbf{cannot} look inside to see how it calculates the answer. Our goal is to discover properties of $f(x)$ using the minimum possible number of queries (questions) to the Oracle.
\end{warningbox}

\section{Part A: The Secret Mechanism (IBM Quantum Composer)}
\textit{Platform: IBM Quantum Composer}

\subsection{Exercise 1: Phase Kickback}
\textbf{Context:} As explained in Lesson \textbf{QC13}, quantum parallelism evaluates $f(0)$ and $f(1)$ simultaneously. But to extract the answer, we need \textbf{Phase Kickback}. This phenomenon occurs when a target qubit is in the state $|-\rangle$. Applying a CNOT gate in this condition transfers a phase change \textit{back} to the control qubit, rather than flipping the target!

\begin{studentbox}
\textbf{Action:}
\begin{enumerate}
    \item Add an \texttt{H} gate to $q_0$ (Control). It is now in state $|+\rangle$.
    \item Add an \texttt{X} gate, then an \texttt{H} gate to $q_1$ (Target). It is now in state $|-\rangle$.
    \item Apply a \texttt{CNOT} gate (Control $q_0$, Target $q_1$).
    \item \textbf{Look at the Q-Sphere} before and after the CNOT.
\end{enumerate}
\textbf{Expected Effect:}
Normally, a CNOT flips the target. But here, the target $q_1$ remains $|-\rangle$. Instead, the phase of the control qubit $q_0$ is flipped from $|+\rangle$ to $|-\rangle$! The "action" kicked back to the control.
\end{studentbox}



\begin{composerbox}
\textbf{Q-Sphere Visualization}\\
Before the CNOT, the dots on the Q-Sphere for the control qubit are Blue (Positive phase). After the CNOT, the dot corresponding to the $|1\rangle$ component of the control qubit turns \textbf{Red} (Negative phase). The target qubit's probabilities did not change at all.
\end{composerbox}

\section{Part B: Coding Deutsch's Algorithm (Qiskit)}

\subsection{Exercise 2: Finding the Function Type}
\textbf{Context:} We are given a function $f(x)$ that takes 1 bit (0 or 1) and returns 1 bit (0 or 1). 
There are two types of functions:
\begin{itemize}
    \item \textbf{Constant:} Returns the same value for all inputs (e.g., always 0, or always 1).
    \item \textbf{Balanced:} Returns 0 for half the inputs and 1 for the other half.
\end{itemize}
Classically, we must check both $f(0)$ and $f(1)$ (2 queries). Quantumly, we can do it in 1 query.

\begin{tcolorbox}[colback=green!5!white,colframe=green!75!black,title=Coding Task: Deutsch's Algorithm]
\begin{lstlisting}[language=Python]
from qiskit import QuantumCircuit
from qiskit_aer import AerSimulator
from qiskit.visualization import plot_histogram

# 1. Setup: 2 Qubits (q0 is input, q1 is ancilla/target)
qc = QuantumCircuit(2, 1)

# 2. Initialization
qc.h(0)           # Put input in superposition |+>
qc.x(1)           # Put ancilla in |1>
qc.h(1)           # Put ancilla in |-> for phase kickback

qc.barrier()      # Visual separator

# 3. THE ORACLE (Example: A Balanced Function)
# We implement f(x) = x. This is a CNOT gate.
qc.cx(0, 1)

qc.barrier()

# 4. Interference and Measurement
qc.h(0)           # Mix the phases
qc.measure(0, 0)  # Measure the input qubit

# 5. Execute
job = AerSimulator().run(qc, shots=1000)
counts = job.result().get_counts()
print(f"Result: {counts}")
\end{lstlisting}
\textbf{Analysis:} 
If the result is \textbf{0}, the function is Constant. 
If the result is \textbf{1}, the function is Balanced. 
Since we used a CNOT (which acts differently for 0 and 1), the function is Balanced, and your output should be 100\% '1'.
\end{tcolorbox}



\begin{deepdivebox}
\textbf{Beyond Deutsch: Why does this matter? (Lesson QC14)}\\
You might think: \textit{"Who cares if a 1-bit function is constant or balanced?"}
Deutsch's algorithm has no direct practical application. However, it proves that quantum interference can solve a problem in $\mathcal{O}(1)$ steps instead of $\mathcal{O}(N)$.
\begin{itemize}
    \item \textbf{Deutsch-Jozsa:} Expands this to $N$ bits. The classical computer needs $2^{N-1} + 1$ queries. The quantum computer still needs exactly \textbf{1 query}. This is an exponential speedup!
    \item \textbf{Grover's Algorithm:} Uses similar Oracle/Phase kickback logic to search unsorted databases quadratically faster.
    \item \textbf{Shor's Algorithm:} Uses these principles to find prime factors, fundamentally threatening modern RSA cryptography.
\end{itemize}
\end{deepdivebox}
