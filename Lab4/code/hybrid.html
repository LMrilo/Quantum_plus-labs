Hybrid Classical-Quantum Computing — VQE & QAOA

75 min Prerequisites: Lab 3.6 NISQ-Era Focus Bridge to Lab 5

Today's quantum computers cannot run the deep circuits required by Shor or Grover on large inputs. The practical solution is hybrid algorithms: short parametric quantum circuits evaluated repeatedly, guided by a classical optimizer. This lab builds the full technical foundation for the finance application in Lab 5.

 The NISQ Constraint

IBM's best QPUs (2025) sustain coherent circuits of roughly 100–300 two-qubit gates before noise dominates. VQE and QAOA are designed around this limit: the quantum circuit is kept shallow while the classical optimizer does the heavy lifting of navigating the parameter space.

Part A: The Parameter-Shift Rule

For any gate $G(\theta) = e^{-i\theta P/2}$ (Pauli generator $P$), the expectation value $f(\theta)$ is a trigonometric polynomial $a + b\cos\theta + c\sin\theta$. Its exact derivative is:

$\partial_\theta f = \tfrac{1}{2}\left[f(\theta + \pi/2) - f(\theta - \pi/2)\right]$

Part C: QAOA for MaxCut — 4-Node Cycle

Graph $C_4$: nodes $\{0,1,2,3\}$, edges $\{(0,1),(1,2),(2,3),(3,0)\}$. Optimal cut = 4 edges (partition $\{0,2\}$ vs $\{1,3\}$). Explore how increasing the number of QAOA layers $p$ improves the approximation ratio and concentrates probability on the optimal bitstrings $|0101\rangle$ and $|1010\rangle$.

From graph to Hamiltonian — derivation

The MaxCut objective counts the number of edges $(i,j)$ where nodes $i$ and $j$ are in different partitions. Assign $x_i \in \{0,1\}$ to each node. Edge $(i,j)$ is cut iff $x_i \neq x_j$, i.e., $x_i + x_j - 2x_ix_j = 1$. So:

$$\text{MaxCut}(x) = \sum_{(i,j)\in E} (x_i + x_j - 2x_ix_j)$$

Substituting $x_k = \frac{1-Z_k}{2}$ (so $x_k=0 \leftrightarrow Z_k=+1$, $x_k=1 \leftrightarrow Z_k=-1$):

$$x_i + x_j - 2x_ix_j = \frac{1-Z_i}{2} + \frac{1-Z_j}{2} - 2\cdot\frac{1-Z_i}{2}\cdot\frac{1-Z_j}{2} = 1 - \frac{1+Z_iZ_j}{2} = \frac{1-Z_iZ_j}{2}$$

Therefore the cost Hamiltonian is:

$$H_C = \sum_{(i,j)\in E} \frac{I - Z_iZ_j}{2}$$

Intuition: $Z_iZ_j$ has eigenvalue $+1$ when both qubits are equal (not cut), and $-1$ when they differ (cut). So $\frac{I-Z_iZ_j}{2}$ equals 1 for a cut edge and 0 otherwise — exactly what we want.

The mixer Hamiltonian $H_B = \sum_i X_i$ creates transitions between bit strings. The $X$ gate flips $|0\rangle \leftrightarrow |1\rangle$, so $e^{-i\beta H_B}$ allows the optimizer to explore different node assignments — without it, the algorithm would stay stuck in its initial state.

Part D: Bridge to Lab 5 — From MaxCut to Portfolio

 The Portfolio Optimization QUBO

In Lab 5 you will minimise a Quadratic Unconstrained Binary Optimization (QUBO) objective over $n$ binary asset variables $x \in \{0,1\}^n$:

$\min_{x \in \{0,1\}^n}\; x^T Q\, x \quad\text{where}\quad Q_{ij} = \lambda\,\Sigma_{ij} - \mu_i\,\delta_{ij}$

($\Sigma_{ij}$: return covariance; $\mu_i$: expected return; $\lambda$: risk-aversion parameter.)

Substituting $x_i = (1 - Z_i)/2$, the objective becomes the cost Hamiltonian of a QAOA circuit:

$H_\text{portfolio} = \sum_{i,j} Q_{ij}\, \dfrac{(I - Z_i)(I - Z_j)}{4} \;+\; \underbrace{\lambda_\text{pen}\!\left(\sum_i \frac{I-Z_i}{2} - K\right)^{\!2}}_\text{budget constraint penalty}$

Connection to this lab:

  • The QAOA circuit structure ($e^{-i\gamma H_C}$, $e^{-i\beta H_B}$) is identical to what you built for MaxCut.
  • The $ZZ$ rotation $e^{-i\gamma Z_iZ_j}$ is implemented with the same CNOT–$R_z$–CNOT decomposition.
  • The classical optimizer (COBYLA or SPSA) is unchanged; only $H_C$ is different.
  • The parameter-shift rule computes gradients for all $(\gamma_k, \beta_k)$ exactly as in VQE.

Part E: Qiskit Python Implementation

Exercises 1–3: PSR, VQE, QAOA Open Colab Notebook 
from qiskit import QuantumCircuit
from qiskit.circuit import Parameter, ParameterVector
from qiskit.quantum_info import SparsePauliOp
from qiskit_aer.primitives import Estimator
from scipy.optimize import minimize

# ── QAOA circuit for MaxCut ─────────────────────────────────────────────────
def build_qaoa(n, edges, p):
    g = ParameterVector('γ', p)
    b = ParameterVector('β', p)
    qc = QuantumCircuit(n)
    qc.h(range(n))
    for k in range(p):
        for (u, v) in edges:          # cost unitary e^{-iγ ZZ}
            qc.cx(u, v)
            qc.rz(2 * g[k], v)
            qc.cx(u, v)
        qc.rx(2 * b[k], range(n))     # mixer e^{-iβ X}
    return qc, list(g) + list(b)

# ── MaxCut Hamiltonian H_C = Σ (I - ZiZj) / 2 ─────────────────────────────
def maxcut_hamiltonian(n, edges):
    terms = []
    for (u, v) in edges:
        zz = ['I'] * n
        zz[u] = 'Z'; zz[v] = 'Z'
        terms += [(''.join(reversed(zz)), -0.5),
                  ('I' * n,                0.5)]
    return SparsePauliOp.from_list(terms).simplify()

# ── Run QAOA ────────────────────────────────────────────────────────────────
edges = [(0,1), (1,2), (2,3), (3,0)]   # 4-cycle C4
for p in [1, 2, 3]:
    qc, params = build_qaoa(4, edges, p)
    H = maxcut_hamiltonian(4, edges)
    est = Estimator()

    def cost(x):
        job = est.run([(qc, H, x)])
        return -job.result()[0].data.evs   # negate: maximise cut

    x0  = [0.5] * (2*p)
    res = minimize(cost, x0, method='COBYLA')
    print(f"p={p}  MaxCut≈{-res.fun:.3f}/4  ratio≈{-res.fun/4:.3f}")
